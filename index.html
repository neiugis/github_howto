Host Your Web Map



  • Go to https://github.com/ and sign up for a GitHub account. Check your email to verify your email account, OTHERWISE you may not be able to publish your web page.
  • In your GitHub home page, click the New button to create a new repository (think repository as a file folder on GitHub used to organize a single mapping project).


  • Next in the Create a new repository page (see below), name your repository. And CHECK ON Add a README file (see image below). Then click Create repository.

  • With the master branch selected, click Add file and then Upload files.

    • RENAME your final map to index.html BEFORE uploading it. This is an important step as GitHub only recognizes index.html as the page to be published.
    • Under Commit changes, enter some text to describe your upload. Make sure you are committing directly to the master branch. Click Commit changes.



      Once the processing is finished, you should see both the README.md and index.html in your repository page.
  • Click the Settings tab, scroll down to the GitHub Pages section. Then select the master branch as the Source and click on the Save button.

  • You are done! Scroll down to the GitHub Pages section again and you will now see your web url.

  • Verify the url and submit it via D2L.
