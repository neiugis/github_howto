Host Your Web Map

We will use the web version of GitHub to host a page for now, which is easier to get started. Whenever you are ready, we will shift to GitHub Desktop. Or you may continue using the web version if you prefer.


  • Go to https://github.com/ and sign up for a GitHub account. Check your email to verify your email account, OTHERWISE you may not be able to publish your web page.
  • In your GitHub home page, go to the Repositories tab and click the New button to create a new repository (think repository as a file folder on GitHub used to organize a single mapping project).


  • Next in the Create a new repository page (see below), name your repository. And CHECK ON Add a README file (see image below). Then click Create repository.

  • With the main branch selected, click Add file and then Upload files.

    • RENAME your final map to index.html BEFORE uploading it. This is an important step as GitHub only recognizes index.html as the page to be published.
    • Under Commit changes, enter some text to describe your upload. Make sure you are committing directly to the main branch. Click Commit changes.



      Once the processing is complete, you should see both the README.md and index.html in your repository page.
  • Click the Settings tab, if you cannot see the "Settings" tab, select the dropdown menu, then click Settings.
  • In the Code and automation section of the sidebar, click Pages. Under Branch, use the dropdown menu to select main and click on the Save button. Then select the main branch as the Source and click on the Save button.

  • To see your published site, under GitHub Pages, click Visit site.

    Note: It can take up to 10 minutes for changes to your site to publish after you push the changes to GitHub. Refresh the site in a few minutes and check if the url appears.

  • Verify the url and submit it via D2L.
